# Nova 与传统 RPA 迁移对照

> 如果你用过 UiPath / 影刀 / 八爪鱼 / Power Automate，这张表帮你快速对应。

## 概念对照

| 传统 RPA 概念 | Nova 对应 | 说明 |
|-------------|---------|------|
| 流程 (Workflow) | **对话指令** | RPA 画流程图，Nova 说一句话 |
| 活动 (Activity) | **Skill** | RPA 的「点击」「输入」对应 Nova 的 Skill |
| 选择器 (Selector) | **自然语言描述** | 「那个蓝色的提交按钮」→ AI 自动定位 |
| 变量 (Variable) | **上下文记忆** | Nova 记住对话中的所有数据 |
| 触发器 (Trigger) | **cronjob / webhook** | 定时或事件驱动 |
| 异常处理 (Try-Catch) | **AI 自主判断** | 出错后 AI 自己想办法重试或换方案 |
| 录制器 (Recorder) | **对话式描述** | 不需要录制，描述你想干什么就行 |
| 编排器 (Orchestrator) | **Hermes 调度** | 多任务自动排队执行 |

## 常见流程迁移示例

### 影刀 → Nova

| 影刀流程 | Nova 指令 |
|---------|---------|
| 打开浏览器→登录淘宝→搜索关键词→提取商品列表→写入Excel | 「帮我在淘宝搜'机械臂'，把前 50 个商品名称和价格整理成飞书表格」 |
| 读取Excel→逐行填入网页表单→点击提交→记录结果 | 「读取这个 Excel，帮我把每行数据填到那个网页表单里提交」 |
| 定时打开邮箱→下载附件→提取PDF表格→汇总 | 「每天早上 9 点检查新邮件，把 PDF 附件里的表格汇总发我」 |

### UiPath → Nova

| UiPath 流程 | Nova 指令 |
|-----------|---------|
| Sequence + Assign + Click + Type Into + Get Text | 「打开飞书文档，把标题改成XXX，内容替换成YYY」 |
| For Each Row + Read Cell + HTTP Request + Write Cell | 「把 Airtable 里每个客户的名字拿去企查查搜，把结果写回来」 |
| Flowchart + Decision + Multiple Branches | 「如果合同金额大于 10 万就走审批流程 A，否则自动通过」 |

### 八爪鱼 → Nova

| 八爪鱼配置 | Nova 指令 |
|-----------|---------|
| 列表页→详情页→字段提取→翻页→导出 | 「爬取这个网站所有产品页，提取名称/价格/描述，存飞书多维表格」 |
| AJAX 加载→等待元素→提取→滚动加载更多 | 「这个页面是动态加载的，帮我滚动到底把所有内容都抓下来」 |

## 迁移清单

| 步骤 | 操作 |
|------|------|
| 1 | 列出你现有的 RPA 流程清单 |
| 2 | 对每个流程，用一句话描述它「做什么」而不是「怎么做」 |
| 3 | 对照上面的 Skill 分类表，看哪些 Nova 已经能覆盖 |
| 4 | 对于 GUI 自动化独有场景，用 `macos-computer-use` + `browser-act` 覆盖 |
| 5 | 复杂多步骤流程，拆成多个 cronjob + webhook 串联 |

## 传统 RPA 无法替代 Nova 的场景

这些是 RPA 做不到，Nova 可以的事：

1. **非结构化理解** — 「这封邮件语气有点急，帮我起草一个安抚回复」
2. **多源信息融合** — 「把小红书上的用户评价和淘宝评论对比分析」
3. **自主决策** — 「这个供应商的报价比市场价高 15%，帮我查一下为什么」
4. **内容生成** — 「根据这周的销售数据，写一份老板汇报 PPT」
5. **AI 判断** — 「这几份合同里哪些条款有法律风险？标红并解释」
