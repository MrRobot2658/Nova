Section 01
传统 RPA 的 三大死结

❌ 死结一：只能处理结构化流程

UiPath、影刀依赖选择器定位界面元素。一旦页面结构变了、数据不是表格、内容需要理解——立刻崩溃。非结构化文档（PDF合同、邮件正文、截图）完全无力。

❌ 死结二：学习成本高、维护噩梦

学一个 RPA 平台要 2-4 周。写一个流程要画几十个节点的流程图。界面一改，整个流程重录。一个企业养 2-3 个 RPA 工程师是标配。

❌ 死结三：跨系统联动靠 API 对接

飞书 / 企业微信 / 钉钉 → 邮件 → 公众号 → 小红书？传统 RPA 的「连接器」数量有限，没开发的平台就接不了。想做一条跨 5 个平台的工作流？先写 5 个 API 对接。

六层架构 · 技术总览

Nova 基于 Hermes Agent 引擎，六层架构从用户输入到平台集成全链路覆盖。不重复造轮子，聚焦 Skill 封装和分发。

层级模块核心能力
L1CLI & 安装器nova 命令 · install.sh 一键部署 · 健康检查
L2Agent Core自然语言解析 · 意图路由 · 上下文记忆 · 多模型路由
L3Skill 系统28 预制 Skill · 办公/采集/引擎/发布四大分组

关键设计：L2 以下全部复用 Hermes 已有能力（Agent 引擎、模型路由、记忆系统、调度引擎）。Nova 的增量价值在 L3（28 个行业 Skill 封装）和 L1（一键安装 + 极简操作体验）。
