## 1. 引言

### 1.1 目的

本文从系统角度描述 Nova 的总体设计：系统边界、模块划分、模块职责与接口、关键数据流、技术选型与非功能性设计。它是连接「产品需求」与「模块详细设计」的桥梁，供研发、测试、运维共同参照。

### 1.2 范围

本文覆盖 Nova **产品运行时**（CLI、Agent 核心、Skill 系统、AI 引擎、调度与执行、集成适配）的概要设计。官网（`site/`）作为分发渠道单独说明，不属于运行时。Hermes Agent 作为依赖的底座，只描述 Nova 如何使用它，不重复其内部设计。

### 1.3 术语

| 术语 | 含义 |
|------|------|
| Hermes | Nova 依赖的底层 Agent 运行时，提供会话、工具注册、模型路由、持久化记忆 |
| Skill | 自包含的能力单元（指令 + 脚本），统一输入/输出/错误处理规范 |
| Profile | Hermes 的运行配置集合，Nova 默认使用 `devops` profile |
| 意图（Intent） | 从自然语言解析出的「目标平台 + 动作 + 输出 + 触发方式」结构 |
| 会话（Session） | 一次自然语言任务的执行上下文，含记忆与中间结果 |
| 编排（Orchestration） | 将一个需求拆解为多个 Skill 并按依赖顺序调度执行 |

## 2. 系统概述

### 2.1 定位

Nova 把 Hermes Agent 包装成「自然语言驱动的办公自动化套件」，替代传统 RPA（UiPath / 影刀）。用户说人话，Nova 自动解析、决策、跨平台执行。

### 2.2 设计目标

| 目标 | 衡量标准 |
|------|---------|
| 零学习成本 | 用户无需学平台、写代码、画流程图，自然语言即可下达任务 |
| 开箱即用 | 安装 ≤ 5 分钟，安装后单条命令即可执行任务 |
| 高可扩展 | 新增一个集成平台 = 新增一个 Skill，不改动核心 |
| 容错自愈 | 单个 Skill 失败可重试或换方案，不导致整个流程崩溃 |
| 长期运行 | 支持定时/事件触发的无人值守自动化，任务间互不干扰 |

### 2.3 设计原则

1. **复用而非自研** — Agent 基础设施（会话/工具注册/模型路由/记忆）复用 Hermes，Nova 只做上层 Skill 封装与分发。
2. **能力单元化** — 每个能力封装为标准 Skill，接口统一，可组合、可替换、可独立分发。
3. **按场景分组而非技术栈** — 模块与 Skill 的组织以「用户想干什么」为轴。
4. **AI 原生** — 非结构化理解、决策、内容生成由 LLM 承担，而非规则引擎硬编码。
5. **无状态执行 + 持久化记忆分离** — 执行过程无状态便于并发，上下文与偏好交由记忆层持久化。

### 2.4 约束与假设

- 运行依赖 Hermes Agent 已安装于 `~/.hermes`，且配置了可用的 AI API Key。
- macOS 为一等平台；Windows 通过 WSL2 间接支持。
- 桌面 GUI 自动化依赖 macOS Accessibility API。
- 不做：低代码拖拽设计器、独立调度引擎、企业级权限体系、私有化部署（详见产品边界）。

## 3. 总体架构

Nova 采用六层架构，自上而下：

```
┌───────────────────────────────────────────────┐
│  L1 接入层   nova CLI · install.sh / .ps1        │  M1
├───────────────────────────────────────────────┤
│  L2 智能核心 意图解析·路由·记忆·工具调度(Hermes)  │  M2 / M3
├───────────────────────────────────────────────┤
│  L3 能力层   Skill 系统（28 个，4 大分组）        │  M4
├───────────────────────────────────────────────┤
│  L4 AI 引擎  LLM · Vision · TTS · 代理            │  M5
├───────────────────────────────────────────────┤
│  L5 运行时   cronjob · webhook · GUI/浏览器执行   │  M6 / M7
├───────────────────────────────────────────────┤
│  L6 集成层   飞书/Google/Notion/邮件/发布平台…    │  M8
└───────────────────────────────────────────────┘
```

层与层之间单向依赖（上层调用下层），同层模块通过 Skill 标准接口协作。

## 4. 模块划分

| 模块 | 名称 | 所属层 | 一句话职责 | 详细设计 |
|------|------|--------|-----------|---------|
| **M1** | CLI 与安装模块 | L1 | 提供 `nova` 命令入口与跨平台一键安装 | [§2](详细设计.md#2-m1cli-与安装模块) |
| **M2** | 意图解析与路由模块 | L2 | 把自然语言解析成意图并匹配 Skill 组合 | [§3](详细设计.md#3-m2意图解析与路由模块) |
| **M3** | 编排调度与记忆模块 | L2 | 按依赖顺序执行 Skill、管理上下文与中间结果 | [§4](详细设计.md#4-m3编排调度与记忆模块) |
| **M4** | Skill 系统模块 | L3 | 定义 Skill 规范、注册发现、生命周期与分组 | [§5](详细设计.md#5-m4skill-系统模块) |
| **M5** | AI 引擎模块 | L4 | LLM 推理、视觉 OCR、TTS、网络代理与模型路由 | [§6](详细设计.md#6-m5ai-引擎模块) |
| **M6** | Runtime 调度模块 | L5 | 定时（cronjob）与事件（webhook）触发，会话隔离 | [§7](详细设计.md#7-m6runtime-调度模块) |
| **M7** | 自动化执行模块 | L5 | macOS 桌面 GUI 与浏览器自动化执行 | [§8](详细设计.md#8-m7自动化执行模块) |
| **M8** | 集成适配模块 | L6 | 统一封装各目标平台的读写与发布 | [§9](详细设计.md#9-m8集成适配模块) |

> 模块 M2/M3 的能力主要由 Hermes 提供，Nova 通过 `devops` profile 配置并在其上扩展；本文按 Nova 视角描述其使用方式与约定。

## 5. 模块间接口与数据流

### 5.1 核心请求生命周期（对话驱动）

```
用户输入「自然语言指令」
   │  ① CLI 透传            (M1 → M2)
   ▼
意图解析  ── 产出 Intent{平台,动作,输出,触发} ── (M2)
   │  ② 意图路由 → Skill 编排计划 Plan[]   (M2 → M3)
   ▼
编排调度  ── 取记忆/上下文 ── (M3 ↔ 记忆)
   │  ③ 按 Plan 顺序调用 Skill            (M3 → M4)
   ▼
Skill 执行 ── 调用 AI 引擎做推理/OCR ── (M4 → M5)
           ── 调用执行器操作 GUI/浏览器 ── (M4 → M7)
           ── 调用集成适配读写平台 ────── (M4 → M8)
   │  ④ 返回结构化结果，写回上下文        (M4 → M3)
   ▼
结果聚合 → 输出（终端 / 飞书 / 文件 …）   (M3 → M1/M8)
```

### 5.2 调度触发生命周期（定时/事件驱动）

```
cronjob 到点 / webhook 收到事件   (M6)
   │  ⑤ 拉起独立 Session，注入预设指令
   ▼
进入 §5.1 的「意图解析 → 编排 → 执行」主链路
   │  ⑥ 执行日志 / 失败重试 / 超时控制    (M6)
   ▼
结果推送 + 记录执行历史
```

### 5.3 模块接口约定

| 接口 | 提供方 → 调用方 | 形态 | 关键字段 |
|------|---------------|------|---------|
| `parseIntent(text)` | M2 → M3 | 同步 | `platform / action / output / trigger / confidence` |
| `plan(intent)` | M2 → M3 | 同步 | `steps[] = {skill, params, dependsOn}` |
| Skill 调用 | M3 → M4 | 标准 Skill 协议 | 入参 JSON / 出参 JSON / error / timeout |
| `infer(prompt, opts)` | M4 → M5 | 同步/流式 | `model / messages / tools / temperature` |
| `act(target, op)` | M4 → M7 | 同步 | `app / selector|nl-locator / action / value` |
| 集成读写 | M4 → M8 | REST/SDK 适配 | `auth / resource / payload` |
| `register(cron|event, instruction)` | M3 → M6 | 异步 | `schedule / event / sessionTemplate` |

### 5.4 全局数据结构

```jsonc
// Intent —— 意图解析输出
{
  "platform": ["feishu"],            // 目标平台
  "action":   "summarize+send",       // 动作
  "output":   "excel",                // 期望产物
  "trigger":  { "type": "cron", "cron": "0 9 * * *" },
  "confidence": 0.92
}

// PlanStep —— 编排计划单元
{
  "id": "s1", "skill": "feishu-doc-create",
  "params": { "scope": "yesterday" },
  "dependsOn": [], "retry": 2, "timeoutSec": 120
}
```

## 6. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| Agent 底座 | 基于 Hermes，不自研引擎 | 复用成熟的会话/工具/模型路由/记忆，聚焦上层价值 |
| 能力封装 | Skill（自包含）而非 Plugin | 可控、易分发、无外部强依赖 |
| 主力模型 | DeepSeek V4 | 中文推理强、价格合理、可控 |
| 模型策略 | 多模型路由 + 故障降级 | 简单任务走 MiniMax（快/省），视觉走 QwenVL，故障切 Qwen |
| 调度 | 复用 Hermes cronjob/webhook | 不另起独立调度器 |
| 平台优先级 | macOS 优先，Windows 走 WSL2 | 目标用户 Mac 为主，降低初期复杂度 |
| 主要竞争对标 | UiPath / 影刀 | 同为「企业级流程自动化」品类，对标最具说服力 |

## 7. 部署架构

```
┌── 用户机器 (macOS / Windows+WSL2) ──────────────┐
│   nova CLI  ──>  Hermes (~/.hermes, devops)       │
│                    │                              │
│   cronjob/webhook ─┘   本地执行：GUI / 浏览器     │
└──────────────┬───────────────────────────────────┘
               │ 出站调用（含代理层）
               ▼
   AI 模型 API (DeepSeek/MiniMax/Qwen·QwenVL)
   集成平台 API (飞书/Google/Notion/Airtable/邮件…)

官网 site/  ──(git push 触发)──>  Vercel 静态托管（与运行时解耦）
```

- 安装：`code/install.sh`（macOS，含 `--dmg` 打包）/ `code/scripts/install-windows.ps1`（WSL2）。
- `nova` 为 `~/bin/nova` 薄封装：`hermes chat --profile devops -- "$@"`。
- 官网与运行时完全解耦，独立部署。

## 8. 非功能性设计

| 维度 | 设计要点 |
|------|---------|
| **性能** | 模型路由按任务复杂度分级；同会话内 Skill 结果缓存到上下文，避免重复拉取 |
| **可靠性** | Skill 级重试（默认 2 次）+ 超时控制；定时任务独立 Session，单任务失败不影响其它 |
| **可扩展性** | Skill 即插即用：放入 skills 目录、符合 SKILL.md 规范即自动注册 |
| **安全** | 凭证交由 Hermes 安全存储；出站可走 HTTP/SOCKS5 代理；最小权限调用集成 API |
| **可观测** | 每次执行记录意图、Plan、各 Skill 耗时与结果；定时任务保留执行历史 |
| **容错** | 选择器失效时由 AI 重新定位（自然语言定位），而非硬编码坐标 |

## 9. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| Hermes 接口/能力变更 | 核心链路受影响 | 通过 profile 与适配层隔离，锁定依赖版本 |
| 模型输出不稳定 | 意图解析/结果偏差 | 多模型路由 + 置信度阈值 + 失败重试 + 关键步骤人工确认 |
| 集成平台 API 变更 | 单个 Skill 失效 | Skill 单元化，受影响面可控，单独升级 |
| GUI 自动化脆弱性 | 桌面任务失败 | AI 自然语言定位 + 重试，必要时降级为浏览器/API 方案 |
| 反爬与网络限制 | 采集类任务受阻 | 代理层 + 限速 + `china-web-data-collection` 策略 |
