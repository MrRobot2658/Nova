# Nova 产品功能

> AI 办公自动化套件。自然语言驱动，28+ 预制 Skill，替代 UiPath / 影刀。

## 产品定位

**一句话**：传统 RPA 说「点这个按钮、填那个框」——Nova 说「把飞书/企业微信/钉钉的消息自动汇总成周报」。

**目标用户**：不需要 IT 部门的业务人员、创业者、中小企业管理者。会用飞书/微信就能用。

**核心差异**：不是「给技术人员用的自动化工具」，而是「给普通人用的 AI 员工」。

---

## 七大核心功能

### 1. 自然语言驱动

传统 RPA 学一个流程要 2-3 天，Nova 10 秒。

```
👤 "帮我把这周飞书文档汇总成 Excel 发给老板"

Nova 自动：
1. 找到飞书文档 → 筛选本周更新的
2. 提取关键内容 → 按日期/项目分类
3. 生成 Excel → 通过飞书发给指定联系人
```

不需要学任何新工具、不需要写代码、不需要画流程图。**说话就行**。

### 2. 跨平台智能联动

Nova 打通了 10+ 个平台，数据能在它们之间自由流转：

```
飞书/企业微信/钉钉 → 公众号文章 → 小红书 → Twitter
邮箱 PDF → OCR 提取 → Notion 数据库 → Airtable 报表
网页数据 → 飞书多维表格 → 定时刷新
```

传统 RPA 每加一个新平台就要装一个 Connector，Nova 的 Skill 机制让新平台接入成本降到写一段自然语言指令。

### 3. 定时 + 事件双引擎

| 触发方式 | 示例 |
|---------|------|
| **每天 9:00** | 生成昨日工作日报发飞书 |
| **每小时** | 检查竞品官网是否更新 |
| **每周一** | 汇总上周飞书文档，生成周报 PPT |
| **webhook** | 客户提交表单 → 自动录入 Airtable + 发飞书通知 |
| **邮件到达** | 新邮件附件 PDF → OCR 提取 → 存档 |

定时任务不是简单的「到点了跑一个脚本」，而是「到点了跟 AI 说一句话，AI 自己完成整个流程」。

### 4. 非结构化数据处理

这是传统 RPA 永远做不到的事：

| 场景 | Nova |
|------|------|
| "这个合同有没有坑？" | AI 读取合同 PDF，标注风险条款并解释 |
| "这 50 封客户邮件情绪怎么样？" | AI 逐封分析，输出情绪分布 + 需要优先处理的 |
| "这张截图里是什么数据？" | QwenVL OCR 识别图片中的表格/文字 |
| "对比小红书和淘宝的同类产品评价" | 同时爬两个平台，AI 对比分析输出结论 |

传统 RPA 处理非结构化数据靠 OCR 硬凑 + 正则匹配，遇到稍微复杂的内容就崩。Nova 的 AI 引擎原生理解文本、图片、文档。

### 5. 桌面 GUI 自动化

不限于浏览器，**macOS 上任何软件都能控制**：

- 操作 Excel 做数据透视
- 操作微信发消息
- 操作 Photoshop 批量处理图片
- 操作 Keynote 生成演示文稿

底层走 macOS Accessibility API，不需要安装任何插件。跟传统 RPA 不同的是：不需要录制操作步骤，描述目标就行——「把这张图片亮度调高 20%，裁成 16:9」。

### 6. AI 自主决策

Nova 不只是执行指令，还能在流程中做判断：

```
流程：每天检查供应商报价
├── 价格正常 → 自动录入系统
├── 价格偏高 > 10% → 自动查市场均价 → 生成议价建议
└── 价格异常 > 30% → 标记风险 → 发飞书告警
```

这种分支决策在传统 RPA 里需要手动画决策树，在 Nova 里是 AI 的默认行为。

### 7. 开箱即用 + 一键安装

```bash
bash install.sh  # 一行命令搞定
nova "你的需求"   # 开始干活
```

传统 RPA 的实施周期：选型 2 周 → 部署 1 周 → 开发流程 2-4 周 → 测试 1 周 = **6-8 周**。

Nova：安装 5 分钟 → 说一句话 → 立刻工作。

## 四大场景矩阵

### 🏢 办公场景

| 场景 | 传统方式 | Nova |
|------|---------|------|
| 日报/周报 | 手动汇总，1-2 小时 | 定时自动生成 |
| 邮件批处理 | 逐封处理 | 批量读取→分类→处理 |
| 文档格式转换 | 手动复制粘贴 | Markdown→飞书/公众号/PPT 一键转换 |
| 会议纪要 | 会后凭记忆写 | 录音→AI 转写→自动生成纪要 |
| 合同审查 | 法务逐条看 | AI 标红风险条款+解释 |
| 发票整理 | 手动录入 | OCR 识别→自动入表 |

### 🕷️ 数据场景

| 场景 | 传统方式 | Nova |
|------|---------|------|
| 竞品监控 | 每天手动刷官网/公众号 | 定时检查→有更新自动通知 |
| 行业数据采集 | 采集工具配置半天 | 一句话描述→自动采集汇总 |
| 裁判文书/法规 | 手动搜索下载 | 批量爬取→分类入库 |
| 客户信息补全 | 企查查逐家查 | Excel 名单→批量查询→回写 |
| 社交媒体舆情 | 手动刷 | 关键词监控→自动汇总 |

### ⚡ 自动化场景

| 场景 | 传统方式 | Nova |
|------|---------|------|
| 表单→数据库 | iPaaS 工具配置 | webhook 自动触发 |
| 审批流 | OA 系统 | 飞书消息→AI 判断→自动处理 |
| 数据同步 | ETL 工具 | 平台 A 变更→平台 B 自动更新 |
| 系统巡检 | 运维脚本 | 定时检查→异常自动告警 |

### 📤 发布场景

| 场景 | 传统方式 | Nova |
|------|---------|------|
| 公众号排版 | 秀米/135 编辑器 | Markdown→自动排版→预览 |
| 多平台分发 | 逐平台手动发布 | 一键发布小红书/视频号/Twitter 等 |
| 内容二次利用 | 重写 | 飞书文档→公众号文章→小红书图文→Twitter 线程 |

## 产品边界（不做什么）

| 不做 | 原因 |
|------|------|
| 低代码拖拽设计器 | 跟「自然语言驱动」冲突，增加复杂度 |
| Windows 原生客户端 | 初期 macOS 优先，Windows 通过 WSL2 跑 |
| 企业级权限管理 | 面向个人/小团队，复杂权限是过度设计 |
| 独立调度引擎 | 复用 Hermes cronjob，不自研 |
| 私有化部署 | 第一批用户不需要 |
